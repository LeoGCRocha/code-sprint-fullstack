# Product

## Register

product

## Users

Students learning data structures and algorithms: beginners and intermediates practicing for coding interviews or academic competition. They are building confidence, not optimizing for rank. They open Code Sprint on a laptop at a desk or coffee shop, often mid-study session. The primary job is: pick a problem, understand it, write code, see if it works.

## Product Purpose

Code Sprint is a competitive programming practice platform. It lets users solve algorithm problems in a timed competition format and in open practice mode. Success looks like a user completing a session, seeing clear feedback on their submission, and feeling motivated to come back.

## Brand Personality

Encouraging, clean, focused. Three words: approachable, sharp, warm.

## Anti-references

- **LeetCode**: dense tables, small text, ultra-utilitarian layout, zero visual hierarchy
- **HackerRank**: corporate feel, heavy institutional branding, form-heavy survey aesthetic
- **Dark/hacker aesthetic**: terminal black, neon-on-dark, "l33t" culture clichés, VS Code cosplay

## Design Principles

1. **One clear task per screen** — every page should communicate a single primary action. Don't let secondary information compete.
2. **Warmth earns trust** — the warm coral + warm sand palette isn't decoration; it reduces the anxiety of a blank editor. Preserve and extend it, never override with cold grays.
3. **Components over one-offs** — Badge, Button, Container exist so every surface speaks the same language. Inline ad-hoc styles are a smell, not a shortcut.
4. **Space as hierarchy** — generous padding and white space separate sections more clearly than dividers or borders. Resist cramming.
5. **Motion signals, not performs** — animation only when it carries information (LIVE pulse, progress fill). Never decorative.

## Accessibility & Inclusion

WCAG AA: minimum 4.5:1 contrast for body text, 3:1 for large text and UI components, keyboard-navigable interactive elements, no color-only state communication.
